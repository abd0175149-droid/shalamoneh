# 🏗 بنية المشروع التقني (Flutter Architecture)

سنعتمد هندسة فصل المزايا (Feature-First Architecture) مع مبادئ الـ Clean Architecture لتسهيل الصيانة والتوسع مستقبلاً.

## هيكلة مجلدات المشروع (`lib/` directory)
```text
lib/
 ├── core/              # المكونات الأساسية المشتركة 
 │    ├── theme/        # الألوان، الخطوط، التنسيقات (Dark Theme)
 │    ├── constants/    # الروابط، النصوص الثابتة، مفاتيح الـ API
 │    └── network/      # إعدادات الاتصال بالخادم (Dio/Http)
 │
 ├── features/          # المزايا الرئيسية (كل ميزة مستقلة)
 │    ├── auth/         # شاشات ومنطق تسجيل الدخول (OTP)
 │    ├── home/         # الشاشة الرئيسية والإعلانات
 │    ├── loyalty/      # الباركود، حساب محفظة النقاط والمكافآت
 │    ├── menu/         # قائمة المشروبات، ومنطق التخصيص السعري
 │    └── locator/      # الخريطة ومواقع الفروع
 │
 ├── shared_widgets/    # المكونات الرسومية القابلة لإعادة الاستخدام
 │    ├── yellow_btn/   # الزر الأصفر الموحد
 │    └── product_card/ # بطاقة عرض العصير
 │
 └── main.dart          # النقطة المركزية لتشغيل التطبيق
```

## 🔄 التكامل والربط الخارجي (Integrations)
* **نظام الكاشير (POS Integration):** بناء Webhooks/APIs للربط مع نظام نقاط البيع الموجود حالياً في الفروع لتحديث النقاط وحالة المنتجات فورياً.
* **الإشعارات:** استخدام Firebase Cloud Messaging لإرسال الإشعارات التسويقية.
